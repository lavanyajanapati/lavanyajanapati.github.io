Project

Real-Time Streaming Pipeline

BTC/USD Live Streaming Pipeline

View on GitHub ↗

End-to-end real-time pipeline ingesting live BTC/USD price data via WebSocket, publishing to Redpanda Kafka with exactly-once delivery guarantees, and processing through Spark Structured Streaming into a Bronze/Silver/Gold Delta Lake architecture on Databricks.

WebSocket → MessageEnricher → Redpanda Kafka → Spark Streaming → Delta Lake

3 partitions · 3 replicas · SCRAM-SHA-256 · 7-day retention

Bronze (raw) → Silver (deduplicated) → Gold (VWAP + OHLCV)

Kafka Producer & Consumer

  • · acks=all — waits for all replica acknowledgements
  • · Idempotent producer prevents duplicate writes on retry
  • · enable.idempotence=true + transactional ID
  • · Consumer group with auto partition rebalancing
  • · isolation.level=read_committed for exactly-once reads
  • · SCRAM-SHA-256 auth over SASL/PLAINTEXT

Spark Structured Streaming

  • · Kafka source with startingOffsets=earliest
  • · Event-time processing via event_timestamp field
  • · Watermark-based late data handling
  • · dropDuplicates() on message ID + event window
  • · Checkpointing to cloud storage for fault tolerance
  • · Trigger interval tuned per layer (micro-batch)

Delta Lake — Medallion Architecture

  • · Bronze — raw WebSocket records, schema-on-read
  • · Silver — deduplicated, validated, typed records
  • · Gold — aggregated VWAP + OHLCV per time window
  • · ACID transactions + time travel via Delta log
  • · Append + merge modes per layer
  • · Optimized with Z-ordering on event_timestamp
